

# PRizm Beauty — デザイン完全移行計画

**美容インフルエンサー × 企業マッチングプラットフォーム**を React/Vite (Lovable) で再構築します。

---

## フェーズ1: デザインシステム構築

Tailwind設定とCSSにPRizmのデザイントークンを実装:
- **カスタムカラー**: `pastel-pink (#FFD6E8)`, `pastel-purple (#E6D5FF)`, `pastel-blue (#D5E8FF)`, `light-gray (#F8F9FA)`
- **グラデーション**: ピンク↔ブルー、パープル↔ピンクのグラデーションパターン
- **エフェクト**: `shadow-soft`, `animate-float`
- **フォント**: Inter + Noto Sans JP（Google Fonts読み込み）
- **favicon**: PRizmアイコンに差し替え

## フェーズ2: 共通UIコンポーネント

アップロードされたコンポーネントを忠実に移植:
- **Button**: グラデーション対応（`gradient`バリアント）、`rounded-full`スタイル
- **Card**: `shadow-soft`ホバーエフェクト付き
- **Input / Textarea / Select / Checkbox**: パープルフォーカスリング
- **FileUpload**: ドラッグ&ドロップ対応、プレビュー表示
- **Modal**: オーバーレイ付きダイアログ
- **Toast**: 成功/エラー/警告の通知コンポーネント
- **Badge**: カテゴリ・ステータス表示用

## フェーズ3: レイアウトコンポーネント

- **Header**: グラデーションロゴ「PRizm Beauty」+ ナビゲーション + LINEログインボタン（緑）
- **Footer**: サービスリンク集 + コピーライト + SNSアイコン
- **InfluencerSidebar**: パステルピンクグラデーションのマイページ用サイドバー（アバター、ナビリンク、モバイルハンバーガー対応）

## フェーズ4: 公開ページ（LP系）

- **トップページ（/）**: HeroSection（グラデーション背景 + フローティングアニメーション）+ サービス紹介3ステップ + CTA
- **キャンペーン一覧（/campaigns）**: 検索・フィルター（カテゴリ、報酬、フォロワー数）+ カード型案件リスト + ページネーション
- **キャンペーン詳細（/campaigns/:id）**: ヒーロー画像 + 案件情報 + 応募条件 + 応募ボタン + パンくずナビ
- **利用ガイド（/guide）**: ステップ説明 + FAQ（アコーディオン形式）+ お問い合わせセクション
- **プライバシーポリシー（/privacy）**: 静的テキストページ
- **利用規約（/terms）**: 静的テキストページ

## フェーズ5: 認証ページ

- **ログイン（/auth/login）**: パステルグラデーション背景 + LINE/Google/Instagramログインボタン + メールログイン（セッションストレージでモック認証）
- **新規登録（/register）**: ステップ式登録フォーム（基本情報 → SNS連携 → カテゴリ選択 → 確認画面）

## フェーズ6: マイページ（インフルエンサーダッシュボード）

サイドバー付きレイアウトで以下のページを構築:
- **ダッシュボード（/mypage）**: 統計カード（応募数・進行中・完了・収益）+ 進行中案件リスト + TODOリスト
- **案件検索（/mypage/campaigns）**: フィルター付き案件一覧
- **応募履歴（/mypage/applications）**: タブ切替 + ステータスバッジ + テーブル表示
- **投稿管理（/mypage/posts）**: 投稿一覧 + ステータス管理
- **プロフィール設定（/mypage/profile）**: アバター変更 + 基本情報編集 + SNS連携 + 銀行口座
- **お気に入り / メッセージ / お知らせ**: プレースホルダーページ

## フェーズ7: ルーティング & 仕上げ

- React Router で全ページのルーティングを設定
- マイページ系はInfluencerSidebarレイアウトでラップ
- モックデータを組み込み
- セッションストレージベースのモック認証でログイン状態管理
- レスポンシブ対応の最終確認

---

> **注意**: この計画はフロントエンドのデザイン移行です。実際のバックエンド（認証・DB・API）は後のフェーズでSupabase/Lovable Cloudを使って実装可能です。
